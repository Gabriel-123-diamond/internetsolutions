1. Fix the Authentication Middleware

Your portal needs a "public" landing page for unauthenticated users (like your phone). Edit app.js (or your main router file) to allow access to the login page even if there is no session.

Find the code that checks for the session and modify it so it doesn't block the login route:
JavaScript

// Example modification in app.js
app.get('/', (req, res) => {
    // If the user has a session, send them to the dashboard
    if (req.session.userId) {
        return res.render('dashboard');
    }
    // If they DON'T have a session, send them to the VOUCHER LOGIN page
    // (Don't return a 403 error here!)
    res.render('portal-login'); 
});
